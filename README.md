# ケアプラン作成支援AIアプリ プロジェクト記録

## プロジェクト概要
ケアマネージャーの業務負担を軽減するための、AIを活用したケアプラン作成支援アプリケーション。
音声入力や基本情報から、Gemini APIを用いて居宅サービス計画書（第1表〜第3表）や家族向けサマリーなどを自動生成し、整合性チェックまでサポートします。

## 最終更新日時
2026年2月28日

## 現在の実装状況（完了済みの機能）

### 1. 共通基盤・UI
*   **アカウントログイン**: デモ用のアカウントログイン画面を実装し、セキュリティ面を強化。
*   **サイドバーナビゲーション**: 各種書類作成画面、AI分析画面へのスムーズな遷移。
*   **状態管理 (`ClientContext`)**: 利用者の基本情報、音声記録、各ケアプラン（第1表、第2表など）のデータをアプリ全体で保持・共有する仕組み。
*   **レスポンシブデザイン**: Tailwind CSSを用いたモダンで使いやすいUI。
*   **取扱説明書（プロモーションページ）**: アプリの価値（時短、ミス防止、対人援助への集中）や主要機能のBefore/Afterを直感的に伝えるランディングページ風のガイド（`Guide.tsx`）を実装。

### 2. AI連携機能 (Gemini API)
*   **音声記録・AI分析 (`VoiceIntake`)**:
    *   面談の音声記録（テキスト）から、AIが自動で「課題」と「ニーズ」を抽出。
    *   抽出したデータをマスターデータ（全体の状態）に反映。
*   **居宅サービス計画書(1) (`CarePlan1`)**:
    *   基本情報と音声分析結果を元に、「利用者及び家族の生活に対する意向」「総合的な援助の方針」をAIが自動生成。
    *   生成されたテキストの編集・保存機能。
*   **居宅サービス計画書(2) (`CarePlan2`)**:
    *   AIが課題（ニーズ）、長期目標、短期目標、サービス内容、サービス種別、頻度をJSON形式で自動生成し、表形式で表示。
    *   生成された内容の編集・保存機能。
*   **週間サービス計画表(3) (`CarePlan3`)**:
    *   第2表のデータを元に、AIが週間サービス計画と日常生活上の活動を自動生成し、マトリクス表に反映。
*   **家族向けサマリー (`FamilySummary`)**:
    *   第1表〜第3表のデータを元に、専門用語を避けた温かみのある言葉で、ご家族向けのサマリーレポートをAIが自動生成。
*   **AI整合性チェック (`AiCheck`)**:
    *   保存されている第1表と第2表のデータをAIが読み込み、内容の矛盾や抜け漏れがないかを自動でチェック。

### 3. テンプレート機能 (`Templates`)
*   現在の利用者のプランデータを「テンプレート」としてローカルストレージに保存。
*   保存したテンプレートを検索し、新しい利用者のデータに適用（コピー）する機能。

### 4. 印刷・PDF出力
*   各書類画面に `@media print` スタイルを適用し、A4サイズ（縦/横）に最適化された印刷・PDF出力機能。

## 今後の展望：他施設への横展開（SaaS化・マルチテナント対応）を見据えて

本ツールは将来的に、自事業所だけでなく**他施設・他法人へのサービス展開（SaaSとしての提供）**を想定しています。
そのため、今後の開発では以下のアーキテクチャ・セキュリティ要件を考慮して進めます。

1.  **マルチテナントアーキテクチャの導入**:
    *   データベース設計において、法人（テナント）IDや事業所IDをキーとし、他法人のデータが絶対に混ざらない（データ分離）セキュアな構造を設計する。
2.  **高度な認証・認可基盤（IAM）**:
    *   Firebase AuthやAuth0などを導入し、法人管理者、事業所管理者、一般ケアマネージャーといった「ロール（権限）ベースのアクセス制御（RBAC）」を実装する。
3.  **セキュリティとコンプライアンス**:
    *   要配慮個人情報（医療・介護情報）を取り扱うため、通信の暗号化（TLS）はもちろん、データベースの暗号化（保存データの暗号化）を徹底する。
    *   監査ログ（誰がいつどの利用者のデータにアクセス・編集したか）の記録機能を実装する。
4.  **カスタマイズ性と拡張性**:
    *   法人ごとに異なる帳票フォーマットの微調整や、地域特有のローカルルールに対応できるような柔軟なテンプレートエンジンを検討する。
